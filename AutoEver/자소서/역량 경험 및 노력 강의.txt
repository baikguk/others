면접과 한큐
면접 준비시 자소서 쉬움(면접고려해서 쓰자)

역량의 구조도 5가지

목표달성능력
문제해결능력
실행력        		 |  ->분석력
     └추진력,리더쉽             |  ->의사소통력

-목표달성능력 : 가장 우선되는 역량 -> 비즈니스의 본질
  조직,개인 모두 목표달성을 위해 있음
     ->성공경험,어려움 극복 경험, 팀웍 경험(어려움)

-문제해결능력 : 목표달성을 위한 필요 역량 ->과정에서 만나는 적
   ex) 목표달성하기에서 문제 봉착
     -> 목표에 도전했던 경험을 써야함 -> 없으면 설명하기 어려움
         해결과정 ex) 아이디어,의사소통능력

-실행력 : 목표달성을 위한 필요 역량 -> 주니어가 가져야할것
       ex) 꾸준히 실행해서 달성했던 경험
             실행력을 위해선 추진력과 리더십
               혼자의 경우 ->추진력
               협업의 경우 -> 리더쉽 

-분석력 : 실행력을 갖기 위해 선행되는 역량
          ex )수리/정보수집력, 기획/계획력, 창의력
              특정 데이터를 분석 경험, 의사소통으로 문제해결경험
-의사소통력 : 실행력을 갖기 위해 선행되는 역량
          ex) 협상,설득력, 공감능력, 스피치 능력
--------------------------------
 분석력(digging력)에 대한 실수
   흔한 실수 2가지 1. 분석툴만 강조 -> 툴을 사용해서어캐분석함?
                         2. 수치의 부재
		  숫자 or 과정 무조건 있음 -> 점점 깊게
                            수치적으로 before/after, 객관적 근거

  분석력 설명 3가지 포인트
    1. 문제정의(조직문제) 후 해결하기 위한 분석
    2. 분석의 기준 
	어떤 기준으로 분석했는지
    3. 나의 가설을 말하는것
	확률이 높은 나의 가설로 분석 시작
	가설 근거 필수
 -> 무엇을 분석했는지보다 분석을 통해 어캐 해결했는지가 포인트
         ㄴ> 분석력을 설명하기 위한 과정을 쓰는것
---------------------------------
역량과 경험 연결시키기
  의사소통 역량(협상,설득력, 공감능력,없으면 비지니스 못함) ->문제해결을 위해
  의사성공 유형 
	1. 정보의 누락을 채워넣는 해동
  	  -> 애매하게 알려줘서 상대방이 모름
	  -> 누락정보 찾아서 개선사례 -> 성공경험으로

	2. 정보의 왜곡사례
	  -> 사람마다 생각하는게 다르기 때문
	  -> 같은 개념,어휘를 다른 관점으로 해석될수도
	  -> 물어보고,확인하고,오류 잡아라

	3. 멘붕 방지 사례
	  -> 기한 못지킴, 안 도와줌 -> 피드백,격려로 돌파사례 성공경험으로

 	-> Best는 내부 부서와 협력으로 어려움극복사례
   		ex) 인턴, 알바에서
----------------------------------
취업필살기 구조도
   지원 직무의 유사경험 ->  성공경험(가장중요)  -> 인사이트

유사경험 : 지원 직무,산업과 동일한 고객을 만나봤는가
              현직자를 만나봤는가
              프젝을 해봤는가
성공경험 : Before/After 명확(나로인해)
              수치의 변화
              무에서 유로 (대단한게 아니여도)
	  경쟁우위 증명(입상/수상)
인사이트 :
              성공의 원리
              유사,성공 포장지
              실패경험 -> 교훈이 있어야함
              구루의 인사이트(성공한사람의 조언)

필살기 : 면접관이 뽑아야하는 이유를 언어로 전달
     수치화 시키는 프레임 3C4P
		
필살기(유사,성공경험) 1분자기소개 2개
빌살기(성공경험만)    면접에서 3개 쓰기
밉살기(성공경험인데 애매한) 성공인데 규모가 작은거
  
-----------------------------------------
3C4P 개념
  논리잡고 구조화하기 위해 (수치 정확성은 상관없음)
  
작성원칙
  1. 경험에서만
  2. 형용사 제거,
  3. 논리구조 탄탄히(퍼즐 형식으로 갖다 붙이기)
  4. 설명안되는건 무조건 제거

3C
  Customer(고객) 
	1. 혜택을 받는 대상
	2. 대상이 필요로 한 것
	  -> 만조시킨 대상은?
		1차 : 실제 고객
		2차 : 협력부서,다른 팀, 회사 전테
		3차 : 상사, 교수, 동료, 팀원(고객이라 가정)
			ex) 오류 발견 -> 팀원들에게 좋음
  Company(자사)
	1. 내가 속했던 곳
	2. 본인(팀)의 목표
	3. 본인(팀)의 문제 혹은 기회상황
	4. 팀 내에서 나의 역할
	  -> 나의 상황
		소속집단에서 나의 역할, 내가 목표했던거

  Competitor
	1. 조사한 대상
	2. 조사 후 적용한 내용
	  -> 논문 조사를 통한 기준,방식,실험방법

4P
Product를 완성하기 위한 나머지 3가지 
 3C에서 문제점 찾기
  Product(상품)
	1. 결과
	2. 결과의 의미
	  -> 내 경험의 결과물은 머지
  Place(위치)
	1. 문제의 해결점 발견
	2. 위치와 채널
	  -> 경험에서 문제가 머였고 무엇을 해결했지?
	  -> 해결포인트 찾기
  Price(가격)
	1. 비용 절감
	  -> 경혐에서 가성비적인 행동이 머였지
	  -> 같은 시간동안 많이하거나, 같은양을 더적게
	 Price ==Place인 경우 -> 시간을 단축해서 해결 
  Promotion(마케팅)(의사소통)
	1. 알리기 관점
	2. 고객 니즈 반영  
	  -> 다른사람에게 잘 설명하기 위해 머했지
	  -> 지출서, 증빙자료

  왜했어요? -> 3C에서 꺼내~
  어떻게했어요? -> 4p에서 꺼내~
  어캐설득함? -> promotion던져~
  -> 이게 될때까지 그 후 자소서

자소서 (무한 얼마나했어? 를 붙여보자)
  what - product -> 소제목 -> 무엇을해서 어떤결과
  why - 고객,company,competitor ->고객 몇명?
  how - price,place,promotion -> 비용절감 얼마나?
  인사이트로 마무리
