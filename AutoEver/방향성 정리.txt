지도 구축 자동화가 필요한 이유 : 기술이 개발되어도 변화탐지와 지도에 반영에 필요한 시간,사람 써야됨
	-> 자동화를 통해 변화 탐지와 더 빠른 서비스 제공을 위해

지도 구축 자동화를 위한 3가지 역할
  	1. 현실 세계 변화정보 탐지
		-> 영상인식
		-> 정밀지도에서 데이터를 추출
		-> 차량용 사용자에서 인입된 패턴 분석
			
	2. HAE 지도와 비교
		->최종적으로 기존 지도 비교하여 차이점을 추출

	3. 지도 구축 자동화 기술 개발 
		-> 추출된 부분에 대한 자동화 기술 개발하여 지도구축함

 2.3 GIS 데이터 분석 단계
	2.3.1  분석 준비 단계
		속성자료 : 질의, 분류, 일반화		
		도형자료 : 동형화, 포멧일치, 경계 부합, 면적 분할, 좌표설정
			  Edgematching -> Mapjoin(merge)
			  Dangle Point인 Over,Undershoot을 Snap distance를 부여하여 자동 연결-> 속성,위상관계가 정의된 지도에서 Topology error을 gis tool로 찾고 Tolerance(허용오차)를 지정하여 지도 보강

	2.3.2  분석 방법론 단계
		--공간분석
		중첩 : union(OR), intersect(AND) 중첩되는 부분만 남김
		        clip(필요부분만 추출), erase(필요없는 부분 삭제) 
		        buffer : 점,선 기준으로 보간해줌
		공간추정 : 1. 경계선 이용(버퍼) 2. 연속적 및 이산적 변이(베리오그램 모델링)
		지형분석 : DEM, TIN, DTED
			-> DXF파일에서 점,라인의 표고값을 참고해 생성가능
			-> Tin to Dem, Dem to TIN다 가능
			-> TIN이 용량이크나 세밀한 정보 가능
			-> Dem to Contour(등고선)의 interval값을 적용하여 등고선 추출 정도 조절 가능
		연결성분석 : 연속성, 근접성, 관망, 확산
			- 위상구조 : 점,선,면의 연결성, 근접성, 계급성 측정해줌
				-> To Coverage기능이 자동구축해줌
				-> Shape와 dxf파일로 가능 각각 방법은 다름
			  Shape의 경우 export -> to coverage
			  DXF의 경우 Feature class to coverage
			  Build(약한 위상구조 정립 기능)
			    폴리곤의 면적,라인의 시종점,길이 등등 
				-> Coverage파일로 변환시 자동으로해줌
				-> 도형의 변환이 필요없음
			  Clean(강한 위상구조 정립)
				-> 도형의 형태를 정리 및 변경하면서 위상정립
			    라인의 Dangle point로 인해 폴리곤 정립이 안된 경우를 tolerance를 부여해 폴리곤을 만들고 -> null값이였던 면적, 라인 양옆 폴리곤의 값을 지정해줌
		지형분석 : 검색기능, Line to polygon, polygon to line
		측정기능

		--공간통계
