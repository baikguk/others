--- GIS란 무엇인가
GIS(지형정보시스템),(Geographic Information System)
	정보시스템에 지형데이터 추가
	ex) 네이버지도, 구글지도 등
  지리공간적으로 참조가능한 모든 형태의 정보를 효과적으로 정의,저장,갱신,조정,분석을 표현할 수 있도록 설계된 컴퓨터의 HW,SW 및 지리적 자료, 인적자원 통합체

-> 통계, 지도 등 활용이 무궁무진(위치정보 기반 통계 데이터 제공)
-> 공간정보를 다룬다
-> 데이터 획득,처리,저장이관건
-> DB, 컴퓨터 시스템
-> 데이터 가공,처리,요약,질의, 시각화가 관건

강의에 Journal, magazine, Organization 있으니까 필요시 강의정보 참고
USGS(미국 GIS), NSDIm, 국토지리정보원,공간정보포털시스템, 국토연구원,
공간정보교육포털

-- GIS정의
   지형공간자료와 속성자료를 연결하여 처리하는 정보시스템

--GIS 개요도
  3차원현실세계에서 데이터 수집 -> 지형정보(도형(레스터,벡터),속성) -> 자료저장
  -> 자료검색/인출 -> 의사결정을 위한정보제공 -> 지도에 반영

--GIS 구성요소
	데이터 -> 구축/저장에 비용이 많이듬(주된 관건)
	HW -> 입출력,연산,저장
	SW -> 입력,중첩,질의,시각화
	인력 -> 활용하는사람
	method -> 비즈니스 모델에 맞는 방법, 응용 프로그램

--국가 사업이 주도해줌(NGIS 사업)

--GIS 기본기능
  지리정보 수집(사진측량,레이저측량(점 좌표취득),GPS(3면측량->위치좌표)
  지리정보 저장(디지털화, 좌표정보(point), 지도생성,도로(line),polygon,cubic)
  지리정보 관리(질의,정보처리) ex) 수치지도에서 SQL을 통한 건물 찾기
  지리정보 분석 ex)공장 적지선정(땅가격,도로인접, 경사도 등등 고려)
  지리정보 시각화(지상레이저를 통한 3차원 시각화), 바람지도,오존지도 시각화

--GIS 활용분야
	교통정보시스템(TIS),도시정보시스템(UIS),자원정보시스템(RIS)



2강 지형공간정보의 이해

--지형공간정보개요
	도형자료(토지모양)와 속성자료(토지면적,소유주)  --> 지도


--지형정보 획득기술의 종류 및 활용분야
  측량(측지측량, GPS위성측량,디지털사진,레이저스캐닝,원격탐사)
	측지측량 : 지구곡률,중력,기하,물리를 고려한 측량
		->VLBI망,세계측지계(ITRF), 대한민국 경위도(ITRF2000)

	일반측량 : 거리, 각 측정하여 다른 한점의 위치를 결정(전통,응용 측량)
		-> 수준측량: 수준원점으로 높이(약 26m)부터 상대적 높이구함
		    인천의 평균 해수면을 0으로 기준
		-> 일반측량 : 거리를 레이저,전자기파를 활용해 좌표결정
				-> 삼변측량:삼각기준점을 기준으로 거리 측정
				-> 토탈스테이션이 대표적 기기

	GPS위성측량 : 위성에서 수신한 전파로 위치결정
		-> 삼변측량과 유사 GPS위치가 기준, 3개의 위성으로 (x,y,z)결정
		-> 위성시간, 측량시간차(△t)도 고려해서 계산
		-> 미지수 4개 -> 위성 4개 필요
		-> DGPS(동적측량),실시간 동적 GPS측량(VRS,비콘활용)
		-> DGPS 정확도를 높이기 위해 위성4개말고,기지국에서도 사용자가 수신한 위성 4개도 계산하여 오차값을 전송해줌)
		-> 상시 관측소가 있음
		-> 사례 : 위성,항공기 위치결정, 정밀 기준점 측량, Mobile GPS, 자동차 항법, 지도제작 및 GIS 데이터 수집
		★★★GPS-VAN(카메라,레이저측량,GPS수신기 탑재)
			->실시간으로 영상,좌표획득으로 GIS데이터 구축

	사진측량 : 정량,정서(색,형태)도 측량 가능
		->항공사진(수치지형도), 도시 상황 조사, 불법건축물 조사

	레이저스캐닝측량: 레이저파를 대상물에 주사, 반사파 측정하여 3차원 실시간 결정
		->항공레이저 스캐닝,지상레이저 스캐닝 측량
		->반사파 돌아오는 시간측정
		-> 시간측정으로 거리계산(수신자 기준으로 반구를 그릴수 있음
		-> 레이저 쏘는 방향 알면 점 좌표로 알 수 있음
		-> 수신자의 위치(GPS로 앎)
		-> 수치지도, 해안선경계
		-> 수집한 3차원 점을 합쳐 -> 지형(산)모델을 생성

	원격탐사 : 위성탑재 센서를 통해 전자기파를 감지해 정보획득,해석
		-> 0.7m정도의 정확성
		-> 산맥고도 측정
		-> 고해상도 위성영상을 위한 모니터링

	-> 1:1000 : 1cm당 1000cm
	-> 내비게이션,네이버 지도 -> 1:1000, 1:500


3강 지도의 이해
  학습 목표 : 지도와 좌표계

--지도의 정의
	지도란 환경에 대한 도해적(Graphical)표현이다. 자연환경과 인문환경의 속성을 나타내고 전달하는 것으로 실제 현상을 단순화시켜 모델화한 것
	기능적 측면 : 지도란 지표상에 분포되어 있는 현상들의 상화관계, 입지, 속성 등에 관한 정보를 저장,전달,분석하기 위한 수단으로 고안된 것
	->수치지도,영상지도,통계지도,2차원지도,3차원지도 등등

--지도의 구성요소
	사용자에 따라 지도 양식, 배치 및 구성이 달라짐
	축적 : 거리 표현에 있어서 비율 정도(1:5000,1:1000,1:500)
		 축적의 표현
			분수표현
			설명적표현
			스케일바
			면적축적
	방위 : 지도상 도북은 진북방향
	범례 : 심볼의 의미 포현(라벨)
	정보원 및 처리방법 : 획득방법, 처리방법
	제목 : 주제와 독자를 고려하여 결정
	투영법 : 실세계의 면적,거리,방향,형상을 지도에 투영하는 방법
	제작자및 제작일,인덱스,지도방안(neat line),부분확대 지도
	-> 일반화,단순화,추상화
	
--지도데이터의 특성
	샘플링(속성데이터): 실세계는 이론상 무한대의 복잡성과 자세함을 가짐
		->완벽한 디지털화는 불가능
	실현을 위한 샘플링 수단을 도입 -> 일정한 관측방법에 의해 구분지어 관측
		->샘플링을 통해 디지털환경에서 현실 세계의 모습을 재현,분석함
	샘플링 기법:
		단순무작위 샘플링(simple random sampling)
		-> 모집단에서 자료가 뽑힐 가능성이 같다는 전제하에 샘플링수행
		-> 모집단에 대한 사전지식없을때 사용

		계층화 무작위 샘플링(startified random sampling)
		-> 모집단을 같은 성질을 갖는 집단으로 나누고(분류?) 집단마다 단순 무작위 샘플링 수행 -> 다수 뿐만아니라 소수로 구성된 집단도 대표하는 장점 

		계통적 무작위 샘플링(systemetric random sampling)
		-> 모집단의 수(N)을 원하는 샘플의 개수(n)으로 나눅 1~N/n의 정수중 한개(i)를 무작위 추출하며 매 i번째 샘플로 취하는 방식 -> 모집단이 무작위로 정렬되어 있다는 전제조건 하에서 유용	

		군집무작위 샘플링(group ranme samplling)
		-> 광범위한 지역에 대한 직접적인 현장 샘플링 필요할 경우 사용함
		-> 모집단을 군집으로 나누고 군집을 무작위 방식으로 선발하여 선발된 군집 안에서 무작위로 자료 취득

		다단계무작위 샘플링
		->위단계를 조합하여 샘플링함

	샘플링 고려사항:
		크기(경제성과 정확성 tradeoff)
		간격
		시기
		기법

	심볼화(도형데이터) : 위치,방향,거리,이동,기능, 점,선,면, 3차원 객체를 심볼화

	심볼화의 방법
		명목척도(Nominal order) 내용을 기반한 정보 분류(도로,강,학교)
		서열척도(Ordinal order) 1차선,2차선 순위에 의해 심볼화(굵기,표현)
		간격척도(Interval order) 온도의 심볼화(레전드)
		비율척도(Ration order) 비율,배율 데이터 

--지도의 분류
	일반도 : 지형도, 지적도,해도,네비게이션 지도
	주제도 : 도형표현도,유선도, 점묘도, 단계구분도(농지,시가지,관강지)
	축적:대축적지도(1:10,000이상),중축적지도(1:10,000~100,000),소축적지도(100,000이하) -> 지도가 자세할 수록 대축적임 
	제작방법
	
	수치지형도, 수치표고모델(DEM), 수치영상지도(항공,위성영상), 3차원수치지도(구글어스, V-world)

좋은 지도란?
	지도는 인간의 도해적 의사소통임으로 시각화, 정보전달, 직관, 신뢰도가 중요
	
	좋은 지도를 위한 고려사항
		지도제작 동기, 목표, 
		지도 사용자 배경지식, 사용처 고려
		지도 구성을 위한 데이터
		사용자가 얻는 정보가 무엇인가
		사용자가 원하는 정보 상세함과 정확성
	


4강 지도의 좌표계( GIS에 있어서 매우중요)
  학습목표 :  좌표계, 기준타원체, 투영법

--지도의 위치표현
	상대적 좌표를 통해 위치파악, 거리 표현
	좌표는 기준점으로 부터 상대적을 얼만큼 떨어졌는가
		
좌표계의 정의와 종류
--좌표계 정의
	좌표란 공간상의 한 물체 또는 한 점의 위치는 일반적으로 좌표로써 표시된다. 위치란 어느 좌표계에 있어서 다른 점들과 어떤 기하학적의 상관관계를 갖는가를 의미하는 것으로, 일반적으로 그 좌표계의 특정점 또는 특정선으로으로부터의 길이와 방향을 매개로 하여 표현된다. 이떄 어느 좌표계의 기준이 되는 고유한 점을 원점(origin), 매개가 되는 실수(길이,방향 등)을 좌표(Coordinate)라 한다.
	-> 원점 : 지구중심

	좌표계 혹은 자리표계는 스칼라 순서쌍을 이용해  n차원 공간의 각 지점을 표현하는 방법을 말한다. 여기서 스칼라는 보통 실수를 말하나, 경우에 따라서 복소수나 다른 일반적인 환(ring)의 원소를 말하기도 한다.
	
--좌표계 종류
	수직선 : 1차원 좌표계
	직교좌표계 : 2차원 좌표계 혹은 3차원 좌표계(z 자전축, xy평면 적도)
	극좌표계 : 평면의 위치를 각도,거리로 나타냄( r,Θ)
	-> 지구를 기준으로한 좌표계가 바람직
	-> 지구를 기준좌표로 어캐 사용할까

--지구 형상의 표현
	물리적 지표면 : 반경(64000km) 타원체
		-> 복잡하고,불규칙적이라 힘듬
	지오이드 : 지표면의 70%를 차지하는 해수면의 평균을 육지까지 연장
		-> 어디에서나 중력방향에 연직인 등포텐셜면으로써 해발고도의 기준면임
		-> 인천평균해수면 기분 -> 지오이드 기준임
		-> 불규칙적인 것을 평평하게 하기위해 해수면을 이용 -> 편리함

	지오이드 특성
		평균해수면을 육지로 연장시킨 가상면
		해수 표면은 물리적으로 평형임으로 지구중력방향과 수직
		지구중력은 지하물질의 밀도 분포와 지형에 따라 굴곡을 갖기 때문에			->지구타원체에 비해 바다에서는 낮고, 육지에서는 높다

	지오이드 결정방법
		지표면의 수많은 점에서 중력 또는 potential을 측정
		절대중력과 상대중력을 측정
		측정된 중력을 해면 위의 중력으로 환산하고 기준타원체와 비교하여 중력이상을 구함
		중력이상을 지오이드高로 환산하고 이를 통해 지오이드 구함
	-> 전세계적 지오이드모델 : EGM96
	
	지구타원체
		지역에 따라(준거타원체) 지오이드 면에 가장 적합한 내접타원체로 수학적으로 결정할 수 있는데 이를 지구타원테 or 기준타원테라 함
		->지오이드 대신에 기준면으로 사용하며 지표면 위의 모든 점의 위치를 결정할 수 있음
		-> 이를 글로벌적으로 통일하기 위해 GRS80(국제공인 타원체, 대한민국 기준타원체)-> WGS84(GPS 좌표체계) 타원체를 만듬

	지구타원체의 기하학적 성분
		자오면 : 지구의 회전축(극축)을 포함한 평면 -> 적도면과 직각
		자오선 : 자오면과 지구타원체의 교선 -> 영국의 그리니치 천문대를 지나는 자오선을 본초자오선이라 함
		평행권 : 적도면과 평행한 평면과 지구타원체와의 교선
		묘유선 : 임의 점에서 수직선(법선)을 포함하며 자오면과 직교하는 평면을 묘유면이라 하며 묘유면과 지구타원체와의 교선이 묘유선
		경도(longitude) : 그리니치 천문대를 지나는 자오면과 임의 점의 자오면이 이루는 각
		위도(lagtitude) : 임의점에서 타원체애 대한 법선이 적도면과 이루는각(타원체 이기때문에 법선이 지도중심을 통과하지 않음)
		-> 각 나라별 지오이드에 맞는 타원체가 달랐음(WGS84, GRS84 매우유사)
						
	지오이드와 지구타원체의 형상비교
		연직선편자 : 기준타원체에 수직인 선과 지오이드 면에 수직인 연직선의 각도 차이, 연직선편차가 최소가 될때 가장 이상적인 지구 타원체
		
--지도의 좌표계
	지도상에 표현되는 지표 위의 많은 점들의 상대적 위치는 동질의 좌표계에 의하여 측정이 이루어 졌을 때 비로소 결정이 가능함
	-> 좌표계 설정하고 측량해야 위치 표현이 가능하다는 듯

	다양한 좌표계가 사용될 수 있으나 어느 것이 더 효율적인가는 사용목적에 따라
	ex) 기준 좌표게: 
		3차원 직각좌표, 측지좌표, 극좌표, 2차원 평면직각좌표

	3차원 직각좌표(지심좌표)
		원점 - 타원체의 중심(지구질량중심)
		x축  - 본초자오선과 적도면이 만나는 지점방향의 연장선
		y축  - x축으로부터 적도면을따라 동쪽으로 90도 위치의 방향
		z축  -  타원체의 회전축

	측지좌표(φ,λ,h)(지리좌표)
		위도(φ 파이 latitude) 임의점을 통과하는 타원체 법선과 적도면이 이루는 각
		경도(λ 람다 Longitude) 그리니치 자오면과 임의점을 통과자오면이 이루는각
		높이(h 높이): 임의점의 법선을 따라 측정된 타원체(WGS84->극좌표의 기준 타원체,구글맵)로부터 거리
	-> 대륙간의 측지측량(VLBI),인공위성측량, 우주개발에서는 지심좌표
	-> 일반적 지도(경위도) 측지좌표 사용

	극좌표
		거리와 방향으로 위치표시
		방향각(β, direction angle) : 어떤 한점에 있어서의 좌표원점 자오선과 평행선인 선과 어떤 방향이 이루는가
		방위각(α, azimuth) : 어떤 한 점에 있어서의 자오선방향과 다른 점을 지나는 선분 방향이 이루는 수평각

	2차원 평명직각좌표
		직각좌표 (X,Y) 
		원점 : 투영중심(지도의 중심)(한국 38도 127.5도)
		지도는 2차원이기 때문에 사용됨

--지도투영
	3차원 지구를 2차원 지도를 만들기 위해 축적을 바구고 형태를 변환시키는 과정
	비틀림 현상을 최소화하여 곡면을 평면으로 옮기는 방법

	투영결과의 3가지 조건
		투영 전후에 대한 점들 간의 거리비가 일정 -> 등거투영
		투영 전후에 대한 형상의 면적비가 일정 -> 등적투영
		투영 전후에 대한 임의의 두 선분이 이루는 각이 동일-> 등각투영
	-> 근데 이 3가지 모두 충족 불가	
		-> 타원체 지구를 평면상에 옮기는데 반드시 1가지 이상의 왜곡발생
		-> 왜곡 처리 방법에 따라 많은 종류의 지도 투영법이 존재

	등각투영
		지도상 많이 사용되는 투영방법
			-> 형상이 변화하지않아 정형도법, 상사투영법이라 함
	
	등적투영
		지역간 면적비를 유지하면서 평면지도에 투영하는 방식
			->지도 상당수가 비등적투영임

	등거투영
		지구타원체 상에서의 거리관계를 지도상에서도 유지
		항공로, 상권의 범위 나타낼때 적합

	투영면의 형태에 따른 분류
		원통도법 : 원통면에 투영 (지구중심이 광원)
		원뿔도법 : 원추면에 투영(지구중심이 광원)
		방위도법 : 평면에 투영(평면에 내접한 점의 반대가 광원)
		->투영이란게 프로젝트빔 느낌임
		-> 지도에 빛을 쏴서 원통면에 어케 표현되는지가 원통도법

	Mercator  투영
		적도면 축적이 1이고 멀어질수록 축적이 급격히 증가
		원통도법

	TM 투영
		가우스크뤼거 투영법
		중앙자오선에서 축적이 1, 멀어질수록 축척증가 
		Mercator 투영 원통을 수평으로

	UTM 투영
		TM과 동일, 적용지역의 중앙자오선 축적을 = 0.9996으로 통일
		지구전체를 경도 6도 간격으로 분할 60개 컬럼(양쪽 +-3도씩 투영)
		180W 자오선에서 동쪽으로 1-60번호 부여
		남위도 80도에서 북위 80도 까지 8도간격으로 분할
		->대한민국 51(120~126경도,123이 중앙자오선) ,52 zone(126~132)

	UPS 좌표계
		UTMP 좌표계의 양극 지방의 좌표를 표시하기 위한 좌표계

	대한민국
		기준타원체 : GRS80
		측지원점 : 위도 37도 16분 33초, 경도 127도 3분 15초
		투영법 : UTM-K(한국형 UTM 좌표계)
			TM으로 하고 축적계수 0.9996
			투영원점의 경위도 : 127도 38도
			-> 좌표의 음값을 없애기 위해 X + 2백만m, y+백만m

--측량의 기준
	사용하는 좌표계에 따라 달라짐

	GPS측량
		3차원 직각좌표계, 측지좌표를 사용함으로 WGS84 기준
		-> 지구질량 중심이 원점

	2차원 직각좌표
		국지적 지역 평면측량(X,Y), 수준측량(z)의 좌표계를 분리하여 측량함
		-> 측량의 기준이 각각 다름
		평면측량 기준 : 측지원점 = 경위도원점 (수원시 팔다구 원천동 111)
			측지원점 : 기준타원체(GRS80) 지오이드가 일치되는 점

			삼각점 : 측지원점 기준으로 전국적으로 분포
	
		표고기준 : 지오이드
			수준원점 : 남구 용현동 253번지(인하대 ㅋㅋ)
				-> 26.6871m(인천만의 평균 해면 위)
			수준점 : 수준원점 기준으로 전국으로 분포

			cf) GPS는 WGS84타원체 기준으로 높이	
		
		
5강 GIS 데이터 모델
  벡터와 래스터 모델(공간데이터), 수치지도(속성데이터)의 이해

  현실세계는 다양한 형태의 공간객체로 구성되며 위치,형태,속성 맟 객체간 관계성이 존재
  공간객체는 형태,위치,관계 등을 점,선,면 등의 단순화된 기하학적 상징물로 표현,
속성은 문자와 상징물로 지도로 표현
  최근에는 지표의 공간사상을 수치화 시켜 표현-> 컴퓨터에 지형공간자료 표현 및 저장을 위해 벡토와 래스터 모델을 사용

 사상(Entity)
	공간상에서 나타날 수 있는 대상물 또는 사건, 즉 현상 중에서 일정 원칙에 의해 경계를 분리하여 속성과 위치를 측정할 수 있는것
	Entity type(사상타입)은 유사한 사상을 집합시킨것
	ex) 도로,건물

 대상물(Feature)
	사상을 GIS 자료로서 DB 내에 표현한 것을 대상물이라고 함
	같은 유형의 대상물끼리 모아놓은걸 대상물 클래스(feature class)
	ex) 지망도,국도,고속도로 등은 대상물이여 도로망도는 대상물 클래스

 공간객체(spatial object)
	공간현상을 사상의 집합으로 인식하여 디지털 방식으로 표현한 것. 
	건물,도로,하천과 같은 분리된 사상은 각각 데이터베이스 내에 공간객체로서 저장

	-> object가 feature보다 큰 개념
	-> 사상이란 자연현상이고, 대상물과 공간객체를 통해서 GIS DATA로 묘사됨
	-> 공간을 사상의 모임으로 간주하여 사상의 특성을 속성정보로, 위치를 특정 좌표계에 기준한 좌표값으료 표현하여 현실세계 묘사


객체타입
  공간객체를 구성하는 원요소임

	0차원 : Point
	1차원 : Line
	2차원 : 면
	3차원 : 점,선,면을 이용해 표현
	
  0차원 객체 타입 : 점(point)
	지리적 위치를 표현하는 객체
	사상점(entity point) : 위치표현 ex) 건물, 빌딩
	라벨점(label point) : 지도 or 차트상 대상물을 표시하는 문자
	면점(area point) : 면내의 한점-> 면의 해당하는 속성을 표현
	절점(node) : 위상학적인 교차점 또는 끝점으로서 위치 표현

 1차원 객체 타입 : line
	선분(line segement) : 두점을 잇는 선, 선의 최소 단위
	연속선분(string) : 다수의 연결된 선분으로 곁가지가 없으며 자체교차 또는 다른 연속선분과 교차할 수 있음
	호(ark) : 수학점 함수로 정의되는 곡선
	링크(link) : 두 절점을 잇는 선으로서 방향을 지닐 경우
	방향링크(directed link) : 방향링크는 네트워크 분석에 주로 이용
	사슬(chain) : 방향을 지닌 다수의 선분 또는 호로서 양 끝점이 절점으로 구성
	링 계열 객체 타입 : 교차되지 않는 다수의 선분, 호, 연결선, 사슬 등으로 구성된 폐합된 고리로서 경계선만을 표현


 2차원 객체 타입 : 면(area)
	면 내부(interior area) : 경계선을 포하치 않는 순수한 면 내부
	화소(pixel) : 영상을 구성하는 최소의 단위로서 보통 밝기 값을 지님
	격자셀(grid cell) : 비교적 일정한 간격을 지닌 면요소로서 연속표면의 표현에 적합


 집합적 공간객체
	0,1,2차원의 객체타입이 집합으로 공간객체를 구성할 때 이를 집합적 공간객체라 함
	디지털 영상(digital image) 화소에 의한 행렬
	격자(grid) 격자셀로 이루어진 행렬, 수치고도모영(DEM)
	자료층 : 특정 주제의 사상들로 이루어진 자료집합
	격자형 자료층 : 같은 격자 또는 디지털 영상에 의한 하나 or 다수의 자료층
	네트워크 : 2차원 객체가 없는 그래프


--레스터와 벡터

  레스터 데이터 모델
	cell이라는 불리는 균일한 형태의 연속된 이산형 그래픽 요소들로 구성된 grid망을 사용하여 실세계를 규칙적으로 공간분할(tessellation) 한것으 ㄹ의미
	-> 래스터는 분석기능이 벡터 GIS 데이터 모델에 비해 풍부함
	-> 레스터 GIS데이터 모델은 자료의 척도,유형,특성에 따라 다양한 시각화가 가능

  Grid
	행과 열로 구성된 셀의 모임
	셀의 모양은 정사각형,직사각형,육각형 등 다양하지만 일반적으로 정사각형

  위치참조(addressing)
	셀의 행열을 사용해 상대적 위치 결정할 수 있음
	기점이 되는 셀에 절대좌표를 부여해 좌표계를 부여할 수 있으

  셀크기
	지도화될 피룡가 있는 최소 사상의 1/4이 이상적
	셀크기를 줄인다해도(resample) 벡터 자료나 위성영상의 원래 해상도보다 나은 정확도를 얻을 수 없음
	벡터자료로 부터 그리드를 생성할때, 고해상도는 선명하지만 저장용량이 큼

  셀이웃
	일정한 규칙에 의해 인접해 있는 셀의 집합

  존(ZONE)
	동일 속성값을 갖는 연속된 셀들의 집합

  지역(Region)
	각각의존에서 연결되어있는 셀들의 집합

  --레스터의 점선면

	점
	  벡터모델에서 X,Y이지만 래스터에서는 하나의 셀로 표현
	  e) 우물, 논의 주인(전주) 등이 하나의 셀로 표현

	선
	  벡터모델에서 X,Y의 순서화된 리스트로 표현되지만, 래스터에서는 공간적으로 인접한 셀의 체인으로 표현
	  ex) 하천, 도로 등이 셀의 체인으로 표현	

	면
	  동일한 공간적 특성을 가지는 셀들의 집합체
	  ex) 토지이용, 영향권역,zone, area

 --래스터 파일 형식

--벡터 데이터 모델
  레스터의 단점을 극복하고, 공간객체를 보다 잘 묘사하기위해
  점,선,면 형태로 공각객체 표현 및 위치가 x,y,z 좌표로 기록
  선은 순서를 가지는 점들의 연속적인 집합으로 면은 점과 선을 이요하여 표현가능
  객체가 크기와 방향성 보유
  객체 별 데이터 관리 가능
  위상구조(Topology) 형성에 유리
	-> Topology란 공간객체가 갖는 상대적인 위치 관계
  정밀한 관리가 필요한 시스템에 사용
  자료 갱신 및 유지관리가 쉬움
  데이터 생성이 복잡하고 어려움

  벡터 위상모델(Topology)
	위상수학은 공간의 위치관계, 가까움을 다루기 위하여 만들어진 수학분야
	공간상 존재하는 객체들은 서로 간에 기하학적인 관계성을 갖으며 이를 위상기하학, 위상구조라함
	각 공간객체 사이의 관계를 인접성(adjacency),연결성(connectivity),포함(Containment)등의 관점에서 묘사
	인접성 : 관심대상 사상의 좌측과 우측에 어떤 사상이 있는지를 정의
	연결성 : 특정 사상이 어떤 사상과 연결되어 있는지를 정의
	포함 : 특정사상이 다른 사상의 내부에 포함되느냐 혹은 다른 사상을 포함하느냔 정의

  위상 모델 구성 ★★★
	노드와 링크로 구성
	노드 : 두 개의 선이 교차하는 지점으로 선의 양 끝점 또는 선상에 주어진 특정한 지점, 예를 들어 도로망, 거주지역의 경계 교차지점 등은 대표적인 노드를 형성
	링크 : 두 개의 노드를 연결하는 선

  위상 모델 테이블
	면 위상테이블 : 숫자로 확인되는 모든 면사상을 구성하는 링크를 열거
	노드 위상테이블 : 각 노드에서 만나는 링크를 열거
	링크 위상테이블 : 각 링크를 구성하는 노드, 면을 이루는 링크를 기준으로 하는 면사상의 좌우 위상관계로 각 노드의 출발점과 끝점을 열거

6강. 수치지도의 이해
--수치지도란?
	수치지도는 지형,지물,지명 등의 각종 지형정보와 관련된 사항을 수치화한 후 전산 시스템을 이용하여 이를 분석,편집, 및 입.출력할 수 있도록 제자된 수치지형도, 수치주제도 등을 말한다

--수치지도 정의

	디지털 매핑
		컴퓨터의 화상처리기술을 이용해 지도에 기재된 정보를 항목별로 분류하여 DB화해서 이용자의 목적에 맞춰 지도를 자유로이 작성하기 위한 기술

	수치지형도
		지형지(feature)에 대한 위치와 형상을 좌표데이터(coordinate)로 나타내어 전산처릭 가능한 형태로 표현한 지도, 일반적으로 자동화된 시스템에 의하여 중,대축적인 지형도나 현황도를 작성하여 수치화한 지도
	
--수치지도 장단점
  장점
	빠르게 제작
	축적의 변환, 투영변호나, 다른 수치지도와의 통합출력이 자유로움
	신축,왜곡, 변형 등이 이루어지지 않는다

  단점
	초기 수치지도 구축에 필요한 시스템의 구입비요,인력 및 조직의 운영, 제도에 많은 투자 필요
	자료를 활영하기 위한 장비와 SW및 전문지식 필요

--수치지도에서의 정보의 표현
	-지형정보
		자연적,인공적인것을 포함한 객체들을 사용자에게 시각적,그래픽적으로 서비스 제공
		점-우물,가로수,가로등
		선-하천,철도,도로,인도
		면-호수,2차원건물
		3차원 객체 - 3차원 건물 등

	-속성정보
		도로,하천,철도 등 모든 수치지도상의 객체들에 대한 X,Y 좌표값 및 다양한 상세정보
		DB를 통한 사용자가 원하는 정보를 속성정보화
		다양한 주제도의 특성에 맞게 DB제작
		속성, 공간정보 DB를 활용하여 고도의 해석을 통해 계획 업무 지원

--수치지형도의 기준
	지표면의 객체의 평면위치와 수직위치를 결정하는 지형측량의 성과표시
	국가기본도는 모든 종류 지도 제작의 기초지도로써 국토지리정보원이 간행하는 전 국토를 통일적.체계적으로 포괄한 일정한 축적의 지형도

	-지형도 기준
		축적 : 1:500, 1:1,000,1:5,000,1:10,000,1:25,000,1:50,000,1:250,000
		준거타원체 : GRS80
		수평기준 : 대한민국 경위도 원점
		수직기준 : 인천항 평균해면, 수준원점
		투영법 : UTM-K(단일중심투영)
		지형표현 : 수평면 정사투영
		표고표현 : 등고선법 이용

--수치지형도의 레이어
	레이어
		기능,표현상 구분이 필요한 내용을 따로 분리해 만들고 저장
		ex) 수치지도에서 등고선,도로,건물 등을 나누어 그림

	수치지형도 Ver2.0기준 레이어 대분류
		교통,건물,시설,식생,수계,지형,경계,주기로 8개
		cf) 교통에 도로경계(A001),도로중심선(A002), 인도(A003), 횡단보도,안전지대 가 있음

--수치지도의 제작방법
  국토지리정보원에서 만듬
   ->측량법에 따른 지도 제작 절차를 따름
   -> 일반적 지형도를 항공사진을 토대로 제작
   -> 한쌍의 항공사진으로 3차원 좌표값 얻음

영상기하보정

--지상기준점 (GCP) -> 공선조건식 상수로 사용
  GCP는 원격탐사에서 영상 좌표계와 지도 좌표계 사이의 좌표변환식을 구하기 위해 사용하는 기준점
  지상기준점을 선정할 때에는 영상과 지도의 두 좌표계에서 공통적인 지점을 선정
  지상기준점은 디지털영상에 정확히 식별이 가능하고 영구적으로 보존되는 지점을 선택
  도로 교차점이나 건물의 모서리 같은 점이 적합하며 계절차나 시기에 따라서 변하는 지점, 예를 들어 강과 다리의 접합 부분등은 부적합
  GCP는 영상 전체에 골고루 분포 시키는 것이 가장 좋으면 GCP의 정확도와 분포에 따라 기하보정 정확돠 크게 영향을 받으므로 신중하게선정
  GCP를 선정하는 방법에는 수치지도에서 독취, DGPS, 현장측량, 기하보정된 영상으로부터의 독취 등의 방법을 사용

 수치영상지도 DEM, TIN의 이해

--DEM
  공간상에 나타나는 지형기복을 2차원 평면에 연속적으로 표현하는 모형을 말하며 수치표고모델이라고 함
  지표면의 표고를 추출하고 이를 일정한 구조로 조직화 하여 컴퓨터 시스템에서 수치적으로 지형을 표현하는 수치지형모형

  DTM(Digital Terrain Model) -> 지형만 표현
  DSM(Digita Surface Model) -> 건물,빌딩등과 함께 표현
  DGM(Digital Ground Model)
  DTED(Digital Terrain Elevation Data)
   등의 용어와 함께 혼용
  
 DEM 제작에서 가장 중요한 원자료는 항공사진이며 위성영상이나 수준측량 등도 많이 이용

  DEM은 지구의 표면이나 대상물 표면을 디지털 형태로 나타낼 수 있기 때문에 도로계획, 지형 및 시설물 해석, 강 형태선(form line)등에 활용

--DEM 형태
  래스터, grid 형태로 표현

--DEM 수집
  지상측량, 레이더표고측량, Lidar, 항공사진, 원격탐사 영상, 등고선 등을 통해 수집 제작됨 -> 레이저측량이 제일 효율적
    -> 등고선으로부터 DEM 생성하는 것은 대부분 상용 소프트웨 등에서 지원
    -> 기대 최대 정확도 등고선 간격의 1/2정도
    -> 항공사진 -> 가장정확함

--TIN(Triangulated Irregular Network)
 불규칙하게 분포된 높이 값을 가지는 점사상으로부터 3차원 지형을 묘사하기 위해 개발된 데이터 모델

 TIN은 포인트, 포인트를 연결하는 선분, 선분들로 이루어진 삼각형의 모자이크로 구성되며 이를 삼각형이 이루는 평면이 3차원 지형의 표면이 된다.
  
 TIN을 구성하는 점을 선택하는 알고리즘에는 Fowler and Little, VIP(very important point), Drop Heauristc이 있다

 선정된 포인트들에 대해 Delaunay Triangulation 알고리즘을 활용하여 삼각형을 형성

  Delaunay 삼각형이란 세 포인트를 연결하여 만들어진 삼각형 안에 다른 포인트가 포함되지 않도록 만들어진 삼각형을 의미
	-> 즉 세점으로 이루어진 외접원을 그리고 그 외접원이 다른 점을 포함하지 않는 경우를 활용하여 삼각형을 형성함
