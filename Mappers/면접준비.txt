	4. DB 역량
		- DB index 사용이유와 장단점
			검색성능 향상 
			실질적 향상은 index, 카디널리티, Selectivity 요소를 고려해 인덱스 생성해야함
			장점 : 빠른 검색 성능
			단점 : 인덱스 저장 공간 관리
			         데이터 삽입, 수정 삭제 시 업데이트

			         미사용 인덱스 성능 저하시킴	
		- 트랜잭션 설명
			DB 상태를 변화시키는 논리 작업단위
			여러 연산 가능 but 하나라도 실패하면 전부실패	

		- ACID : 트랜젝선 안전 수행 보장을 위한 성질
			원자성 : 트랜잭션 연산은 모든 연산이 완벽히 수행되어야함, 하나라도 실패하면 전부실패
			일관성 : 트랜잭션은 유효한 상태로만 변경될 수 있음
			고립성 : 트랜잭션 동시 실행 시 다른 트랜잭션에 영향 받지 않아야함
			내구성 : 트랜잭션이 커밋된 이후에는 시스템 오류가 발생하더라도 커밋된 상태를 유지되는 것을 보장해야함(비휘발성)

		- 트랜잭션 격리 수준
			트랜잭션 격리 수준은 고립도와 성능의 tradeoff 조절
			READ UNCOMMITTED : 다른 트랜젝션에서 커밋되지 않은 내용도 참조할 수 있음 
			READ COMMITTED : 다른 트랜젝션에서 커밋된 내용만 참조할 수 있음
			Reapetable read : 트랜잭션 진입하기 이전에 커밋된 내용만 참조할 수 있음

		- CAP이론과 Eventual Consistency
			Cap 이론은 분산 환경에서 모두를 만족하는 시스템이 없다
			Consistenty(일관성) : ACID의 일관성과 다르게 모든 노느가 같은 시간에 같은 데이터를 보여줘야함
			Availability(가용성) : 모든 동작에 대한 응답이 리턴되어야함
			 Partition Tolerance(분할 내성) : 시스템 일부가 네트워크에서 연결이 끊기더라도 동작해야함

			Eventual Consistency
				모두에게 Consistency를 보장하지 못하지만 결과적으로는 언제가는 Conssistency가 보장됨

	Server
	  ㄴ 네트워크 지식
	  ㄴ Spring 지식
		- IOC(제어역전) : 프로그램의 흐름을 개발자가 제어하지 않고 Spring워크 내부에서 결정된대로 제어함
			
	  	- DI에 대해 : Spring 프레임워크에서 지원하는 IOC 형태로 클래스 사이의 의존관계를 빈 설정 정보를 바탕으로 컨테이너가 자동연결

		- 스프링에서는 스프링 컨테이너(Application Context)를 이용하여 설정 정보를 생성, 등록하고 필요한 객체를 생성자 혹은 setter를 통해 주입

		- Spring Bean이란 IOC 컨테이너 안에 들어있는 객체로 필요할때 컨테이너에서 사용함, Bean어노테이션으로 객체를 컨테이너에 등록함
		
 			
		- 스프링 Bean 생성과정
		  객체생성 -> 의존 설정 -> 초기화 -> 사용 -> 소멸 과정의 생명주기를 가지고 있습니다. Bean은 스프링 컨테이너에 의해 생명주기를 관리하며 빈 초기화 방법은 @PostConstruct, 빈 소멸에서는 @PreDestory사용
		  생성객체를 스프링 빈을 등록할 때 ComponentScan을 이용하거나 @Configuration의 Bean어노테이션을 사용해 Bean 설정파일에 직접 등록

		-스프링 Bean의 Scope이란
		  빈 스코프는 빈이 존재할 수 있는 범위를 뜻하며 싱글톤, 프로토타입, request, session이 있음
		  ㄴ 싱글톤
			기본 스코프로 컨테이너 시작과 끝까지 유지되는 가장 넓은 스코프
		  ㄴ 프로토타입
			빈의 생성과 의존관계주입까지 관여하고 더는 관리하지않은 스코프
		  ㄴ request
			웹 요청이 들어오고 나갈때까지 유지하는 스코프
		  ㄴ session
			웹 세션이 생성 종료될때까지

		-IOC 컨테이너의 역할은
		  애플리케이션 실행시점에 빈 오프젝트를 인스턴스화하고 DI 후에 최초로 앱을 기동할 빈 하나를 제공해준다

		-DI 3가지방법과 차이		
		  1. 생성자 주입
			생성자 호출시점에 딱 1번만 호출하며 필수적이고 불변하는 의존관계에 사용
		  2. Setter 주입
			수정 가능성이 있는 의존관계에서 사용하며 스프링빈을 선택적으로 등록
		  3. 필드 주입
			@Autowired를 통해 주입
			프레임워크없이 작동힘듬
			Autowiring과정은 컨테이너에서 타입을 이용해 의존 대상 객체를 검색하고 할당할 수 있는 빈 객체를 찾아 주입한다.

		-@Bean, @Component 어노테이션 설명과 둘의 차이
		  @Component : 개발자가 작성한 class 기반으로 인스턴스 객체를 1회 싱글톤으로 생성->@Controller,@Service,@Repository는 모두 @Component이며 실행시점에 자동으로 의존성 주입

		@Bean 개발자가 작성한 method 기반으로 메서드에서 반환하는 객체를 인스턴스 객체로 1회 싱글톤으로 생성
